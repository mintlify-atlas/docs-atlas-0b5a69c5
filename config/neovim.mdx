## Overview

This Neovim configuration uses [Lazy.nvim](https://github.com/folke/lazy.nvim) as the plugin manager and includes full LSP support for multiple programming languages, intelligent code completion, and modern development tools.

## Plugin manager

The configuration uses Lazy.nvim for plugin management. It's automatically bootstrapped on first run:

```lua .config/nvim/init.lua
local lazypath = vim.fn.stdpath("data") .. "/lazy/lazy.nvim"
if not vim.loop.fs_stat(lazypath) then
  vim.fn.system({
    "git",
    "clone",
    "--filter=blob:none",
    "https://github.com/folke/lazy.nvim.git",
    "--branch=stable",
    lazypath,
  })
end
vim.opt.rtp:prepend(lazypath)

require("init")
require("lazy").setup("plugins")
```

## Completion

The configuration uses nvim-cmp with multiple sources for intelligent code completion:

```lua
cmp.setup({
  snippet = {
    expand = function(args)
      require('luasnip').lsp_expand(args.body)
    end,
  },
  mapping = cmp.mapping.preset.insert({
    ['<C-p>'] = cmp.mapping.select_prev_item(cmp_select),
    ['<C-n>'] = cmp.mapping.select_next_item(cmp_select),
    ['<C-y>'] = cmp.mapping.confirm({ select = true }),
    ["<C-Space>"] = cmp.mapping.complete(),
  }),
  sources = cmp.config.sources({
    { name = 'nvim_lsp' },
    { name = 'luasnip' },
  }, {
    { name = 'buffer' },
  })
})
```

## Editor settings

Core editor configuration from `.config/nvim/lua/set.lua`:

```lua
vim.opt.nu = true              -- Line numbers
vim.opt.rnu = true             -- Relative line numbers

vim.opt.tabstop = 2            -- Tab width
vim.opt.softtabstop = 2
vim.opt.shiftwidth = 2
vim.opt.expandtab = true       -- Use spaces instead of tabs

vim.opt.smartindent = true     -- Smart indentation
vim.opt.wrap = false           -- No line wrapping

vim.opt.swapfile = false       -- No swap files
vim.opt.backup = false         -- No backup files

vim.opt.hlsearch = false       -- Don't highlight search results
vim.opt.incsearch = true       -- Incremental search

vim.opt.termguicolors = true   -- True color support
vim.opt.scrolloff = 8          -- Keep 8 lines visible when scrolling
vim.opt.colorcolumn = "80"     -- Show column at 80 characters
```

## Color scheme

The configuration uses **Rosé Pine** as the default color scheme with transparency enabled:

```lua .config/nvim/lua/plugins/colors.lua
require('rose-pine').setup({
  disable_background = true,
  styles = {
    italic = false,
  },
})

vim.cmd.colorscheme("rose-pine")
```

Alternative color schemes are also included but not active:
- **Catppuccin Mocha** - Warm, pastel color scheme
- **Tokyo Night Storm** - Dark blue theme

<Info>
  The configuration disables the background color to allow terminal transparency to show through.
</Info>

## Diagnostic configuration

Diagnostics are shown in floating windows with rounded borders:

```lua
vim.diagnostic.config({
  float = {
    focusable = false,
    style = "minimal",
    border = "rounded",
    source = "always",
    header = "",
    prefix = "",
  },
})
```

<Note>
  All LSP servers are automatically installed on first launch through Mason. No manual installation required.
</Note>

## File structure

The Neovim configuration is organized as follows:

```
.config/nvim/
├── init.lua              # Entry point, loads Lazy.nvim
├── lazy-lock.json        # Plugin version lockfile
└── lua/
    ├── init.lua          # Core initialization
    ├── set.lua           # Editor settings
    ├── remap.lua         # Keybindings
    ├── plugins.lua       # Plugin list
    └── plugins/          # Individual plugin configs
        ├── lsp-config.lua
        ├── telescope.lua
        ├── treesitter.lua
        ├── colors.lua
        └── ...
```

<Tip>
  After deploying the configuration, run `:Lazy sync` in Neovim to install all plugins.
</Tip>
