## Overview

[Picom](https://github.com/yshui/picom) is a lightweight compositor for X11 that provides window transparency, shadows, and fade effects. This configuration is used with i3 on Linux.

<Note>
  Picom is only needed on Linux with X11. macOS has a built-in compositor, and Wayland compositors handle these effects natively.
</Note>

## Configuration location

The configuration file is at `.config/picom/picom.conf`.

## Backend

The configuration uses the xrender backend for compatibility:

```conf
backend = "xrender";
vsync = true;
```

<Info>
  The xrender backend is more stable than glx on most systems, especially with NVIDIA GPUs.
</Info>

## Shadow effects

### Shadow configuration

Shadows are enabled with subtle parameters:

```conf
shadow = true;
shadow-radius = 7;
shadow-offset-x = -7;
shadow-offset-y = -7;
```

This creates soft shadows that appear slightly offset from windows.

### Shadow exclusions

Certain window types don't receive shadows:

```conf
shadow-exclude = [
  "name = 'Notification'",
  "class_g = 'Conky'",
  "class_g ?= 'Notify-osd'",
  "class_g = 'Cairo-clock'",
  "_GTK_FRAME_EXTENTS@:c"
];
```

## Transparency

### Opacity settings

Inactive windows are slightly transparent:

```conf
inactive-opacity = 0.95;
frame-opacity = 0.9;
inactive-opacity-override = false;
```

| Setting | Value | Purpose |
|---------|-------|--------|
| `inactive-opacity` | 0.95 | Inactive windows at 95% opacity |
| `frame-opacity` | 0.9 | Window borders at 90% opacity |
| `inactive-opacity-override` | false | Don't override app-set opacity |

<Tip>
  Subtle transparency helps distinguish active from inactive windows without being distracting.
</Tip>

## Fading

Fading is disabled for snappier window transitions:

```conf
fading = false;
fade-in-step = 0.03;
fade-out-step = 0.03;
```

<Accordion title="Why is fading disabled?">
  Fading animations can feel sluggish on slower systems. Disabling them provides instant window transitions, which many users prefer for a responsive desktop experience.
</Accordion>

## Corners

Rounded corners are disabled:

```conf
corner-radius = 0
```

Exclusions are still defined in case you enable rounded corners:

```conf
rounded-corners-exclude = [
  "window_type = 'dock'",
  "window_type = 'desktop'"
];
```

## Background blur

### Blur configuration

A simple 3x3 box blur kernel is used:

```conf
blur-kern = "3x3box";
```

### Blur exclusions

Certain window types are excluded from blur:

```conf
blur-background-exclude = [
  "window_type = 'dock'",
  "window_type = 'desktop'",
  "_GTK_FRAME_EXTENTS@:c"
];
```

<Warning>
  Background blur can impact performance significantly. Consider disabling it on older hardware by commenting out the `blur-kern` line.
</Warning>

## Window type settings

Specific opacity and shadow settings per window type:

```conf
wintypes:
{
  tooltip = { fade = true; shadow = true; opacity = 0.75; focus = true; full-shadow = false; };
  dock = { shadow = false; clip-shadow-above = true; }
  dnd = { shadow = false; }
  popup_menu = { opacity = 0.8; }
  dropdown_menu = { opacity = 0.8; }
};
```

## Performance settings

Optimized for performance:

```conf
mark-wmwin-focused = true;
mark-ovredir-focused = true;
detect-rounded-corners = true;
detect-client-opacity = true;
detect-transient = true;
use-damage = true;
glx-no-stencil = true;
```

| Setting | Purpose |
|---------|--------|
| `mark-wmwin-focused` | Better focus detection |
| `detect-client-opacity` | Respect app-set opacity |
| `use-damage` | Only redraw changed areas |
| `glx-no-stencil` | 15% performance boost (GLX only) |

## Focus exclusions

Cairo-clock never gets focus effects:

```conf
focus-exclude = [ "class_g = 'Cairo-clock'" ];
```

## Log level

Logging is set to warnings only:

```conf
log-level = "warn";
```

<Info>
  Change to `"debug"` if you're troubleshooting issues, but remember to change it back to avoid log spam.
</Info>

## Starting Picom

Picom should be started automatically by i3. Add to your i3 config:

```i3config
exec_always --no-startup-id picom
```

Or start manually:

```bash
picom --config ~/.config/picom/picom.conf
```

## Installation

Install Picom on your Linux distribution:

<CodeGroup>
```bash Ubuntu/Debian
sudo apt install picom
```

```bash Arch Linux
sudo pacman -S picom
```

```bash Fedora
sudo dnf install picom
```
</CodeGroup>

<Tip>
  After deploying the config with GNU Stow, restart Picom with `killall picom && picom &` to apply changes.
</Tip>

## Troubleshooting

<Accordion title="Screen tearing">
  Try enabling vsync in the configuration:
  ```conf
  vsync = true;
  ```
</Accordion>

<Accordion title="Performance issues">
  Switch to a lighter backend:
  ```conf
  backend = "xrender";
  ```
  Or disable blur and shadows.
</Accordion>

<Accordion title="Windows not transparent">
  Check if `inactive-opacity-override` is set to `false` and that the window manager is passing opacity values correctly.
</Accordion>
